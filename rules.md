# UI/UX Design Rules for Gemini CLI

## 1. Visual Hierarchy
- The most important section must be visually dominant (larger size, more spacing).
- Use spacing intentionally to indicate importance and grouping.
- Avoid clutter. Every element must justify its existence.

## 2. Buttons & CTAs
- Primary buttons must have high contrast.
- Avoid ghost buttons for primary actions.
- Every page must have a clear and strong CTA.
- CTA text should be action-oriented (e.g., "Get Started", "Book Now").

## 3. Color System
- Follow the 60-30-10 rule:
  - 60% → Dominant color
  - 30% → Secondary color
  - 10% → Accent color (used for CTAs)
- Maintain consistent color usage across pages.
- Ensure accessibility (sufficient contrast ratio).
- Use tools like https://coolors.co for palette generation.

## 4. Typography
- H1 (Heading): Strong, expressive typography.
- H2 (Subheading): Simpler than H1 but still prominent.
- Body text: Highly readable, simple fonts.
- Maintain consistent font hierarchy and spacing.

## 5. Conversion Optimization
- Reduce friction in user flow.
- Keep content clear and direct.
- Prioritize scannability:
  - Use bullet points
  - Use short paragraphs
- Motivate users with clear value propositions.

## 6. Audience-Centric Design
- Design for the target audience, not the client’s personal preference.
- Content should solve user problems clearly.

## 7. Site Structure

### Navbar
- Must include:
  - Logo (left)
  - Navigation links (center/right)
  - CTA button (right)
- Must be visible on all pages.

### Hero Section
- Clearly explain what the website/product does.
- Be extremely obvious and direct.
- Include:
  - Strong headline
  - Supporting subtext
  - Primary CTA
  - Social proof (if possible)

### Body Sections
Include the following:
- Business/service overview
- Features and benefits
- Examples of past work / portfolio
- Social proof (testimonials, stats)
- Blog (optional)
- FAQ
- CTA section

### Footer
- Include links to all important pages.
- Include company logo.
- Include social/contact links.

## 8. Layout & Design Variation
- Use variation across sections:
  - Layout (grid, split, alternating)
  - Background colors
  - Section height and spacing
- Avoid repetitive design patterns.

## 9. Content Placement
- Place important information higher on the page.
- Follow natural reading patterns (top → down, left → right).

## 10. Development Guidelines
- Create sitemap before development.
- Use wireframes before UI design.
- Maintain consistency across all pages.
- Ensure responsiveness (mobile-first design).

## 11. Accessibility & UX Improvements (Added)
- Ensure text contrast meets accessibility standards (WCAG).
- Buttons must be easily clickable (min height ~44px).
- Avoid tiny fonts (<14px).
- Ensure proper spacing for touch devices.

## 12. Performance (Added)
- Optimize images (lazy loading, compressed formats).
- Avoid unnecessary animations.
- Keep load time minimal.

## 13. Interaction & Feedback (Added)
- Add hover and focus states for buttons.
- Provide feedback on user actions (loading, success, error).
- Avoid dead clicks (every interactive element must respond).

## 14. Common Anti-Patterns to Avoid
- No unclear messaging in hero section.
- No low-contrast buttons.
- No overcrowded layouts.
- No inconsistent spacing or typography.
- No missing CTA on key sections.
